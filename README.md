# Тесты на проверку параметра firstName при создании пользователя в Яндекс.Прилавок с помощью API Яндекс.Прилавок.
- Для запуска тестов должны быть установлены пакеты pytest и requests
- Запуск автотестов выполняется командой pytest create_kit_name_kit_test.py -v

В проекте реализованы:
- функции для отправки запросов к API (post_new_user, post_new_client_kit);
- генерация тестовых данных (get_kit_body);
- автотесты для позитивных и негативных сценариев проверки параметра name.

Структура проекта:
- configuration.py # конфиг: url и пути
- create_kit_name_kit_test.py # автотесты позитивные и негативные
- data.py # тестовые данные
- sender_stand_request.py # функции для создания пользователя и пользователького набора

Чек-лист тестов:
1. Допустимое количество символов (1) - статус-код 201, в ответе поле name совпадает с полем name в запросе
2. Допустимое количество символов (511) - статус-код 201, в ответе поле name совпадает с полем name в запросе
3. Количество символов меньше допустимого (0) - статус-код 400
4. Количество символов больше допустимого (512) - статус-код 400
5. Разрешены английские буквы - статус-код 201, в ответе поле name совпадает с полем name в запросе
6. Разрешены русские буквы - статус-код 201, в ответе поле name совпадает с полем name в запросе
7. Разрешены спецсимволы - статус-код 201, в ответе поле name совпадает с полем name в запросе
8. Разрешены пробелы - статус-код 201, в ответе поле name совпадает с полем name в запросе
9. Разрешены цифры - статус-код 201, в ответе поле name совпадает с полем name в запросе
10. Параметр не передан - статус-код 400
11. Передан другой тип параметра (число) - статус-код 400
